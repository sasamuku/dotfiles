# タイトル・Description の方針タイプと選び方

タイトルと Description は記事の入口であり、流入経路ごとに最適解が違う。SNS で拡散したいのか、SEO で長期流入を狙うのか、業界の問題意識を共有したいのかで方針が変わる。

このリファレンスは方針タイプを整理し、それぞれの長所・短所と、書き手と方向を決めるときの聞き方をまとめる。

---

## 方針タイプの 4 分類

### 1. 思想型（時代性・前提を疑う）

**例:**
- 「〇〇の時代に△△を問い直す」
- 「〇〇必須をやめて△△を導入した」

**長所:**
- CTO・VPoE 層の知的読者に刺さる
- シリーズ感が出やすく、書き手のブランドを積み上げやすい
- 引用されやすい（X / Bluesky 向き）

**短所:**
- 抽象度が高くクリック率は中庸
- 「読まないと中身が分からない」ので SEO 流入は弱め

### 2. 数字訴求型（インパクト重視）

**例:**
- 「〇〇の N% を△△にした話」
- 「△△を 1/X に短縮した〇〇運用」

**長所:**
- SNS でのクリック率が最大
- 数字が読者の好奇心を刺激し、「同じ数字を出したい」と思わせる
- 拡散性が高い

**短所:**
- 数字が独り歩きしやすい
- 「うちは N% じゃないから無理」と短絡される懸念
- 後続記事と並んだとき「数字煽り路線」に固定されるリスク

### 3. ハウツー型（SEO 重視）

**例:**
- 「〇〇の作り方」
- 「△△で〇〇を線引きする」

**長所:**
- 検索流入に強い
- 「自分のチームでもできそう」という持ち帰り感が明確
- ストック型コンテンツとして長期的に読まれる

**短所:**
- ハウツー記事に見えるためインフルエンサー筋の拡散はしづらい
- 話題性は控えめ

### 4. 問いかけ型（共感重視）

**例:**
- 「〇〇に本当に△△が必要ですか？」
- 「××のままで本当にいいですか？」

**長所:**
- 同じ悩みを持つ読者に刺さりやすい
- 引用されやすい問いの形

**短所:**
- 媒体によっては煽りに見える
- SEO は弱め

## 方針を決めるときの聞き方

書き手にいきなり「どの方針がいいですか」と聞くのではなく、**記事の出口（読者にどう届けたいか）** から逆算する。

問いかけの例:

- 「この記事は SNS で拡散させたいですか、SEO で長期流入を狙いたいですか、それとも社内向けに残したいですか」
- 「読者として誰を想定していますか。意思決定層／現場の実務者／業界全体／社内」
- 「同じテーマで後続記事を書く予定はありますか。あればシリーズとして揃えたいトーンは」

書き手が「思想型がいい」「数字を全面に出したい」のような方向を示したら、その方針内で 5〜10 案出す。

---

## 案を出すフォーマット

各案は以下を添えて提示する:

- **タイトル候補**
- **このタイトルが伝えるニュアンス**
- **メリット・デメリット**

最後に **自分の推し** を 1〜3 個選び、理由をつける。

例:

```markdown
### 案 A. 「〇〇の N% を△△にした話」

- **ニュアンス**: 数字の強さで興味を引く
- **メリット**: クリック率最大、SNS 拡散向き
- **デメリット**: 煽り感、後続記事と並んだとき数字頼みに見える
```

---

## 動詞始まりや一行型などの形式制約への対応

書き手が「動詞始まりにしたい」「副題は嫌だ」のような形式制約を出してきたら、その制約の中で再度 5〜10 案を出す。

形式制約の代表例:

- **動詞始まり**: 動詞で言い切る
- **副題なし**: タイトルを 1 行に収める
- **数字必須**: タイトル内に具体数字を入れる
- **疑問文形式**: 問いかけで終わる

書き手の制約は基本的に守る。違和感があれば「〇〇という形式だと△△の理由で△△が難しいので、こういう代替はどうですか」と提案するが、書き手が押し通したらそれに従う。

## Description の書き方

Description は OGP / SEO 説明として表示される。タイトルが思想型なら Description で具体・数字を補う。タイトルが数字型なら Description で文脈を補う。

### 思想型タイトルに合わせる Description

数字 + 主張の構成で具体を補う。タイトルだけだと抽象的で何の話か分からないので、Description で「具体的に何をして、どんな結果が出たのか」を 1〜2 文で示す。

### 数字型タイトルに合わせる Description

タイトルが数字なので、Description は背景や思想を補う。「なぜそれをやったのか」「読者にとっての持ち帰り」を示す。

## NG パターン

### 釣りタイトル

最悪値だけを切り取って改善幅を強調するタイトル。

❌「指標を 〇〇 → △△ に短縮」（〇〇 が長期間の最悪値で、改善幅が実態以上に大きく見える）
✅「指標を 〇〇〜△△ → ×× に短縮」（レンジで誠実に提示）

数字は本文・表と完全に整合する形で書く。

### 副題ベタ貼り

「〇〇した話 — 〇〇から〇〇への取り組み」のような副題は、媒体によっては OGP で切れる。書き手が嫌う場合も多いので、デフォルトは 1 行で言い切る。

### 抽象的すぎるタイトル

「組織の生産性を高めるアプローチ」のような抽象タイトルは、記事の独自性が消える。**記事固有の概念や数字を必ず 1 つ含める**。
